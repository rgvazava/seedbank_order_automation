WRPIS Seedbank Order Automation

Curator Workflow Guide

Overview

This system automates the handling of international GRIN “Foreign New Order Items” reports.

It generates:

  • Filled APHIS permit PDFs
  • Standardized order summary files
  • Structured master email packets

Core Principle: All editing happens in one permanent curator inbox. Timestamped folders are records only.

Daily Workflow

1

Export the GRIN Report

Export the Foreign New Order Items report.

  • Export as CSV (.csv)
  • Do not rename or modify column headers
2

Run the Automation

Drag and drop the CSV file onto:

run_orders.bat
The system processes orders automatically and places them into the correct curator folders.
3

Open Your Permanent Curator Inbox

Location:

outputs/curator_inboxes/<Your_Name>/orders/

Each order folder contains:

  • order_<id>.txt – order summary & taxa list
  • order_<id>_APHIS.pdf – filled APHIS form
  • MASTER email_packet.txt – editable master record
  • OPEN_MASTER_PACKET.bat – opens the master packet

Sending the Permit Email

Important: Status tracking happens ONLY inside the MASTER email packet.
1

Open the Master Packet

Inside the order folder, double-click:

OPEN_MASTER_PACKET.bat

This ensures you are editing the correct master record.

2

Send via Outlook

  • To: Permits (Samantha)
  • CC: Curators listed in the packet
  • Subject & Body: Copy exactly as written
  • Attachment: The APHIS PDF in the same folder
3

Update Status

After sending, edit the master file:

STATUS: NOT SENT → STATUS: SENT

Save and close.

Understanding the Folder System

Permanent Curator Inbox (Source of Truth)

All edits and real work happen here:

outputs/curator_inboxes/

This folder accumulates orders over time. Re-running the program never deletes previous orders.

ALL_INBOX Folder

The ALL_INBOX folder provides quick access to every order across curators.

It contains pointer files only. It does not contain duplicate copies.

Timestamped Snapshot Folders

Located under:

outputs/orders_out/<timestamp>/

These are historical records for auditing or troubleshooting. They are not edited.

System Logic

Primary Curator Selection

The curator with the highest number of items in an order is automatically selected as the primary curator for APHIS processing.

No File Duplication

The system avoids duplicate copies to prevent sync conflicts or version confusion.

Duplicate Protection

If an order already exists, the system will not overwrite SENT master packets.

Rules & Important Notes

  • Do not edit files inside timestamp folders
  • Do not rename order folders
  • Do not move files outside the curator inbox structure
  • Do not modify anything inside DO_NOT_TOUCH/
  • If something looks incorrect, report the Order ID
